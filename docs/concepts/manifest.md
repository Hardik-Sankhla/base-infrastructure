# Discovery Manifest

## What is the Discovery Manifest?

The `DiscoveryManifest` is the absolute source of truth regarding the state of the host environment immediately following the completion of the Discovery Pipeline.

It acts as a dynamic map containing strongly-typed structs generated by individual discovery stages.

## Why a Manifest?

Decoupling the interrogation of the host from the interpretation of those results is critical for stability. The Manifest provides a single immutable snapshot of the system state.

If the discovery pipeline detects an OS, network interface, and installed software, it safely merges them into one atomic object rather than scattering variables globally.

## How it works

The core struct (`internal/domain/models/discovery.go`) looks like this:

```go
type DiscoveryManifest struct {
	Environment string
	Artifacts   map[string]any
}
```

During the final phase of the pipeline, the Engine loops through the `Artifacts` array of each `StageStatus`, and inserts them into this central `map[string]any`. 

The key of the map corresponds to the type of artifact (e.g. `"software"`, `"network"`, `"os"`).

## When is it used?
It is instantiated at the end of the `pipeline.go` execution cycle and immediately passed to the Capability Builder.

## Where is it located?
Defined in `internal/domain/models/discovery.go`.
