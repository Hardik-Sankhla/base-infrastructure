# ADR 0001: Why Go?

## Status
Accepted

## Context
We need a robust language to serve as the core platform runtime for interrogating system environments and executing complex state reconciliation tasks. The language must be deployable across Linux, Windows, macOS, Android (Termux), and embedded systems (like Raspberry Pi).

## Decision
We chose **Go (Golang)** as the core implementation language for the Base Infrastructure platform.

## Consequences

### Positive
- **Static Compilation**: Go compiles to a single static binary. Users do not need to install Python, Node, or Ruby on the target systems to run the bootstrap process.
- **Cross-Compilation**: We can build Windows executables from a Linux CI pipeline seamlessly via `GOOS` and `GOARCH`.
- **Concurrency**: The Discovery Engine heavily utilizes goroutines to run independent system checks in parallel safely.
- **Type Safety**: Structs (like `DiscoveryManifest`) ensure strict data guarantees before executing destructive tasks.

### Negative
- **Plugin Complexity**: Because Go is compiled, plugins cannot be dynamically loaded script files natively without using complex CGO or RPC boundaries. We mitigated this by designing a JSON-RPC STDIN interface for external plugins.
